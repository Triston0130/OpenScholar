MIT OpenCourseWare API Setup Guide

This guide will help you set up Google Custom Search Engine (CSE) to enable full-text search of MIT OpenCourseWare materials in OpenScholar.

Why is this needed? MIT OCW doesn't provide a public API. Without Google CSE, searches return only 1-5 results. With CSE, you can search all 2,600+ courses!

Step 1: Create a Google Custom Search Engine

1. Go to Google Programmable Search Engine

2. Click "Get started" or "Add"

3. Configure your search engine:

  • Search engine name: MIT OpenCourseWare
  • What to search: Specific sites
  • Sites to search: ocw.mit.edu/*

4. Click "Create"

5. On the next page, copy your Search engine ID (looks like: 017643444788069204610:4gvhea_mvga)

Step 2: Get a Google API Key

1. Go to Google Cloud Console

2. Create a new project or select an existing one

3. Enable the Custom Search API:

  • Navigate to "APIs & Services" → "Library"
  • Search for "Custom Search API"
  • Click on it and press "Enable"

4. Create an API key:

  • Go to "APIs & Services" → "Credentials"
  • Click "Create Credentials" → "API Key"
  • Copy the API key

Step 3: Configure OpenScholar

1. Return to OpenScholar

2. Paste your Google API key in the configuration modal

3. Set your CSE ID as an environment variable on your server:

export MIT_OCW_CSE_ID='your-cse-id-here'
Success! Once configured, MIT OpenCourseWare will show a green shield icon and you'll be able to search all MIT courses.

Troubleshooting

No results returned?

  • Verify the API is enabled in Google Cloud Console
  • Check that your CSE is configured to search ocw.mit.edu
  • Ensure your API key has no restrictions or includes Custom Search API

Quota exceeded?

  • Free tier: 100 searches/day
  • Paid tier: $5 per 1,000 queries
  • OpenScholar will fall back to basic search if quota is exceeded
